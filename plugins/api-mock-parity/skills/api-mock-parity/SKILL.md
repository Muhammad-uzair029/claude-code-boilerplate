# api-mock-parity — Mock ↔ contract drift auditor

## Why
Juniors iterate against mocks. The backend evolves. The mock becomes fiction. The client screen ships against a shape prod never returns and starts throwing at runtime. This skill catches that drift before it merges.

## Scope

Sources of client-side mocks:
- MSW handlers under `apps/frontend-ui/src/mocks/**`
- Storybook fixture JSON under `apps/frontend-ui/**/*.stories.@(ts|tsx)` (extract fixture literals)
- Test fixtures under `apps/frontend-ui/**/__fixtures__/*.json`
- Legacy `apps/frontend-ui/mocks/*.json` files

Sources of truth (in precedence order):
1. `packages/contracts/**` — the ideal
2. `apps/backend-api/openapi.{json,yaml}` (checked in) or auto-generated
3. NestJS DTOs (`*.dto.ts`) with `class-validator` decorators
4. Pydantic models (`apps/ai-engine/**/schemas/*.py`)

## How to run

### 1. Enumerate mocks
Walk client mock sources. Extract for each: `{ endpoint | operation-id, method, request-shape?, response-shape }`.

### 2. Enumerate truth
Extract for each backend / ai-engine endpoint: `{ path, method, request-schema, response-schema-per-status }`. Prefer OpenAPI if present; otherwise derive from DTO + decorator + return type.

### 3. Match
- Match by `operationId` if present in OpenAPI + declared in mock.
- Otherwise match by `(method, path-template)`.
- Report unmatched mocks (no backend endpoint) as **stale** — the endpoint was likely renamed / removed.
- Report unmatched endpoints (no mock exists) as **coverage-gap** — informational, not a blocker.

### 4. Diff shape
For each matched pair, normalize to `{ field: {type, optional} }` and diff.

| Situation | Rank |
|-----------|------|
| Field in truth but not in mock (response) | **H** |
| Field in mock but not in truth (response) | **H** |
| Type mismatch | **CRIT** |
| Required field in truth marked optional in mock | **H** |
| Enum member exists in mock but not in truth (or vice versa) | **H** |
| Status code returned by mock not documented in truth | **M** |

### 5. Suggest patch

For each drift, print the minimal JSON patch that would make the mock match. Example:

```json
// mocks/orders/get-order-200.json  — proposed patch
{
- "customer_id": 42,
+ "customerId": 42,
+ "created_at": "2026-07-05T12:00:00Z"
}
```

## Output

`docs/mocks/parity-<YYYY-MM-DD>.md`.

```markdown
# api-mock-parity — 2026-07-05

**Verdict:** in-sync | drift(<n>) | broken(<n>)
**Truth source:** apps/backend-api/openapi.json (autogenerated 2026-07-04)

## Findings
| Mock                                 | Endpoint         | Situation               | Rank |
|--------------------------------------|------------------|-------------------------|------|
| mocks/orders/get-order-200.json      | GET /orders/:id  | field rename (snake→camel) | H |
| mocks/orders/list-orders-200.json    | (no match)       | stale (endpoint removed)   | H |
| — (no mock)                          | POST /refunds    | coverage-gap               | L |
```

## Hard rules
- **Never rewrite mocks silently.** Print the patch; a human applies it.
- **Do not treat coverage-gaps as drift.** They are informational.
- **CRIT type mismatches block the PR.** Silent runtime bug otherwise.
- **Regenerate OpenAPI first if the truth is stale.** Report `WARN — truth source > 30 days old` and stop until the user confirms it is current.
